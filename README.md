# LLM-Based Hierarchical Topic Exploration for Supporting Exploratory Literature Search

**Japanese title:** LLMを用いた階層的トピック提示による探索的論文検索支援

---

## Overview

This system supports **exploratory literature search** by dynamically presenting hierarchical topic structures generated by an LLM. It helps users — especially research beginners — discover relevant research areas and keywords without requiring prior domain knowledge.

When a user inputs a keyword, the system generates parent topics (broader concepts) and child topics (more specific concepts) using GPT-4o-mini, and renders them as an interactive tree visualization with D3.js.

### Background & Motivation

When researchers begin exploring a new field, they often don't know which keywords to search for. Conventional keyword-based search requires users to already know the right terminology — creating a barrier for beginners. This system addresses that barrier by presenting hierarchical topic structures that guide users from a single query toward a well-structured search strategy.

- **Prompt engineering**: Designed a 4-element prompt structure (granularity criteria, prohibited word list, diversity axis, output format) to stabilize LLM output quality
- **Output stabilization**: Implemented automatic JSON retry with exponential backoff and canonicalization-based deduplication to handle LLM output instability

- **Frontend**: Interactive tree visualization using D3.v7 (zoom, expand/collapse, node recentering, Google Scholar integration)

## Key Challenges & Solutions

| Challenge | Solution |
|-----------|----------|
| LLM output instability (invalid JSON, hallucinated topic names) | Auto-retry with exponential backoff + regex-based JSON extraction + canonicalization for deduplication |
| Generic/redundant parent topics (e.g., "科学", "技術") | Server-side filtering with a blocklist of generic terms; generate-many → filter → ensure 2–3 quality parents |
| Topic diversity across expansions | Prompt explicitly instructs diversity along multiple axes (methodology / application / theory); history-aware prompting avoids repetition |
| User study logging for behavioral analysis | Per-subject NDJSON event logging with session management (Flask session + UUID) |

### Prompt Design (4 Elements)

1. **Granularity criteria** — defines the appropriate abstraction level for parent vs. child topics
2. **Prohibited word list** — blocks overly generic terms (e.g., "科学", "技術", "情報科学")
3. **Diversity axis** — instructs the model to cover methodology, application domain, and theoretical background
4. **Output format** — enforces strict JSON schema; combined with auto-retry for robustness

## Architecture

```
User (Browser)
    │
    ▼
[D3.js Frontend]  ←→  [Flask Backend]  ←→  [OpenAI API (gpt-4o-mini)]
    │                       │
    │                  Session Store
    │                  (history, meta cache, trans cache)
    │
    ▼
Google Scholar (external, opened via constructed URL)
```

**Key API endpoints:**
- `POST /expand` — generates parent/child topics for a given node
- `POST /describe` — returns term description and abbreviation (short/full/desc)
- `POST /translate` — translates Japanese terms to English for Scholar search
- `POST /log_ui` — logs user interaction events (NDJSON per subject)

## Experimental Results

- **Participants**: 8 users (research beginners)
- **Method**: Within-subject comparison (system vs. baseline keyword search)
- **Statistical test**: Wilcoxon signed-rank test
- **Result**: Statistically significant improvement in system usability (p < 0.05)
